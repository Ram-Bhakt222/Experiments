1. Starting and stopping the dashboard

First-time launch

Double-click Run Dashboard.bat in the dashboard folder. The first run builds a .venv and installs requirements (≈2–3 min). After that, every launch is fast.

Or from PowerShell:

cd "C:\Users\ram\Desktop\programming projects (other)\Experiments\dashboard"
.\"Run Dashboard.bat"

Open http://localhost:8900. Log in as admin with the SHARED_SECRET you set in .env.

Stop

Ctrl+C in the terminal that launched it. Or just close the window — the dashboard isn't a system service.

2. The interface, in one sentence

The home screen has a grid of pipeline tiles at the top and a live job table below. Click a tile to open a small dialog, drop files / fill inputs, hit Run. The job appears in the table; status auto-refreshes every 2 seconds. When status flips to completed, an Open button appears — that links straight to the output file.

3. Every tile, explained

audio

Accepts.mp3 .m4a .wav .flac .ogg (any ffmpeg-readable audio or video)
Produces<stem>.txt (timestamped), <stem>.srt, <stem>.json

video

AI Video Studio Job

What it isBridge to your existing AI Video Studio. Doesn't reimplement video work — it forwards.
InputsStudio station name (e.g. ltx-t2v, hedra-avatar, silero-vad, ffmpeg-concat) + JSON of station inputs.
Examplestation = ltx-t2v
inputs_json = {"prompt":"sunset over a yoga studio","duration":5}
video

Video → Clips

Rebuild of your original cut_clips.py. Same CSV mode preserved + four new modes.

AcceptsOne video + (optional) one timestamps CSV
Modes timestamps — drop a CSV with type,start,end,label (original cut_clips.py behavior)
equal — split into N equal segments
fixed — consecutive N-second windows (e.g. 60s reels)
scene — ffmpeg scene-detect, cap at N clips
chapters — run whisper, split at largest silence gaps
EncodingLossless ffmpeg -c copy by default (fast, ~1s drift on non-keyframes). Set reencode in inputs for frame-perfect cuts.
Producesclips/clip_##.mp4 per segment + _manifest.json

4. Where outputs land

Every job gets its own folder:

storage/outputs/<job_id>/
  ...the outputs...

The Open button in the job table opens the primary file (the manifest, the markdown, the txt — whatever the pipeline pointed output_path at). You can browse the whole folder via the file explorer for siblings (e.g. the SRT next to the TXT, every clip in clips/).

Job state lives in storage/jobs.db (SQLite). Survives restarts.

5. Logins and multi-user

Two modes:

Quick (default) — single admin account whose password = your SHARED_SECRET in .env.

Multi-user — create users.json with bcrypt hashes. Generate hashes like this:

cd "C:\Users\ram\Desktop\programming projects (other)\Experiments\dashboard"
.\.venv\Scripts\python.exe -c "import auth; print(auth.hash_password('YOUR_PASSWORD'))"

Then paste hashes into users.json per the format in users.json.example.

7. Recipes — chaining tiles

Muhammad Call → 60-sec reels

Drop the .mp4 on Video → Clips, set mode=fixed seconds=60. Wait. Each clip lands in storage/outputs/<job>/clips/. Open the manifest to grab the list.

Hotel call audio → searchable knowledge

(1) Drop each hotel .mp3 on Audio → Transcript. (2) Copy all the .txt outputs into one folder. (3) Run RAG: index a folder pointed at that folder, collection hotel_calls. (4) Query via your Cockpit Qdrant tooling.

Fathom session → SOAP note + email draft

(1) Drop the Fathom transcript on Session → SOAP + Action Steps. (2) The job stages the file. (3) Have Claude run the wombhealth-session-processor skill against the output folder — it'll produce the two .docx files and the email draft.

Long video → chapter clips → b-roll reel

(1) Video → Clips with mode=chapters n=8. (2) For each chapter you want polished, open Studio and run silero-vad on it (silence-cut). (3) ffmpeg-concat them back. (4) Optionally topaz-upscale for 4K.

8. Troubleshooting

"Port 8900 in use" or browser shows nothing

Edit .env, change DASHBOARD_PORT. Or kill whatever's on 8900:

Get-NetTCPConnection -LocalPort 8900 | Select-Object OwningProcess
Stop-Process -Id <PID> -Force

"401 Auth required"

Login user is admin; password is your SHARED_SECRET from .env (until you create a real users.json).

Ollama tile says "connection refused"

Ollama isn't running. From a terminal:

ollama serve

And in another terminal pull a model:

ollama pull llama3.1:8b-instruct-q4_K_M
ollama pull nomic-embed-text

Studio tile says "no job id"

AI Video Studio isn't running at 8780. Launch it from Desktop\AI video gen\studio\Run Studio.bat.

RAG: "folder not found"

The path you typed must exist on the host (the Windows machine running the dashboard), not on a network share that isn't mounted. Use copy-paste from File Explorer's address bar.

Whisper: "no module faster-whisper"

First-run requirements install probably failed (often a temporary network error). From the dashboard folder:

cd "C:\Users\ram\Desktop\programming projects (other)\Experiments\dashboard"
.\.venv\Scripts\activate
pip install -r requirements.txt

FFmpeg not found (Video → Clips fails)

FFmpeg must be on PATH. Install once:

winget install Gyan.FFmpeg

Restart the terminal so the new PATH is picked up, then relaunch the dashboard.
